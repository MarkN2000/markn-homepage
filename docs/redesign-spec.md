# サイトリニューアル仕様書

markn2000.com を一から作り直すための設計ドキュメント。
(ステータス: **確定版** / 2026-07 議論により決定)

## 1. リニューアルの目的

- 記事を**普通のMarkdownファイル**として書けるようにする(専用表記の全廃)
- スマホでも読みやすいレスポンシブデザイン
- 読み込み・動作の軽いサイト(クライアントJSを最小化)
- **何年も保守できる**構成(依存関係を最小化し、放置耐性を最大化)
- デザインは zenn.dev を参考にしたカード型・固定ライトテーマ

## 2. 技術選定(確定)

| 項目 | 選定 | 理由 |
|---|---|---|
| SSG | **Hugo** | 単一バイナリで npm 依存ゼロ。バージョン固定で何年後も同じビルドを再現可能。画像処理・ハイライト・リモート取得を内蔵 |
| CSS | **素のCSS** | Tailwind/PostCSS/npm を全廃。カスタムプロパティによるデザイントークン方式 |
| JS | **vanilla JS 最小限** | 検索絞り込み・リンクコピー・YouTube遅延読込・トップへ戻る程度。ライブラリなし |
| フォント | **システムフォント** | Google Fonts(Noto Sans JP)の読み込みを廃止。ヒラギノ / Noto Sans JP / 游ゴシックのOSスタック |
| ホスティング | Cloudflare Pages(継続) | Pages Functions(`functions/`)は廃止 |

### 保守方針

- Hugo のバージョンをリポジトリに明記して固定(CI でも同バージョンを指定)
- `package.json` / `node_modules` / `postcss.config.js` を撤廃。ビルドは Hugo バイナリ1個で完結
- README にビルド手順を1コマンドで記載

## 3. 記事フォーマット(純粋Markdown)

```yaml
---
title: "記事タイトル"
date: 2026-01-01T00:00:00+09:00
tags: ["Resonite", "作ったもの"]
description: "OGP・一覧に使う説明文"
thumbnail: "thumbnail.webp"   # 任意
emoji: "⚡"                    # 任意。thumbnail がない場合のカバー絵文字
series: "アバター解説"          # 任意。シリーズ記事の前後ナビを自動生成(§6)
draft: false
---
```

### URL自動変換の規則(このサイトの中核仕様)

**前後に空行のある、行に単独で書かれたURL**(=Markdownとして独立した段落のURL)だけをビルド時に変換する。GitHub/Zennと同じ感覚。

| URLの種類 | 変換結果 |
|---|---|
| YouTube | 軽量埋め込み(サムネ+再生ボタンのみ先行表示。クリックで本物のプレイヤーを読込) |
| X(旧Twitter) | ビルド時に oEmbed で本文取得 → 静的な引用カード。取得失敗時は「Xでポストを見る」シンプルカードにフォールバック |
| その他 | OGPリンクカード(§5) |

- 文中に書いたURL・`[テキスト](URL)` 形式のリンクは変換しない(普通のリンクのまま)
- キャプション付き画像は標準記法 `![alt](画像 "キャプション")` → `<figure>`+`<figcaption>`
- 生のHTML(`<br>` 等)はMarkdownの仕様どおりそのまま使える(乱用はしない)
- GFMアラート・読了時間表示は**採用しない**(必要になったら後付け可能)

## 4. デザイン(承認済みモック準拠)

- **固定ライトテーマ**(ダークモードなし。ただし色はすべてカスタムプロパティで定義し将来の追加を容易に)
- 水色基調 + タータン柄アクセント(ヘッダー帯・ロゴ。CSSグラデーションで描画)
- トップ: タグ絞り込み(件数付き) + カードグリッド(3→2→1列) + ページネーション
- 記事: デスクトップは本文+追従目次の2カラム、モバイル1カラム
- カードカバー: thumbnail → emoji+グラデーション の順でフォールバック
- **見出しアンカーリンク**: 見出しホバーで「#」を表示、セクションURLをコピー可能

### デザイントークン

```css
--ground:#EDF4F8;  --surface:#FFFFFF;  --ink:#243642;
--muted:#63798A;   --accent:#2AA3D4;   --accent-deep:#1479AC;
--accent-soft:#E3F2FA;  --border:#DAE6ED;  --code-bg:#1C2B35;
```

## 5. OGPリンクカード(ビルド時取得・画像も自前配信)

- ビルド時に Hugo の `resources.GetRemote` でリンク先のOGP情報を取得し静的HTML化
- **サムネイル画像もビルド時に取得 → リサイズ → WebP化して自サイトから配信**
  (GitHub の camo やはてなブログと同方式。リンク切れに強く、読者のブラウザは外部サイトへアクセスしない)
- 取得失敗時: ドメイン名+ファビコンのシンプルカードにフォールバック。**ビルドは失敗させない**
- 取得結果は Hugo のキャッシュで保持(TTLは実装時に調整)
- クライアントJS・`functions/api/fetch-metadata`・GAS の `ogpApiUrl` は廃止

## 6. 機能一覧

### 引き継ぐ(現行から)

タグ(taxonomy)+件数付きフィルター / ページネーション(12件) / サムネイル・hero画像 /
日付・git lastmod / 関連記事6件 / Xシェア・リンクコピー / OGP・Twitterメタ / schema.org /
sitemap / robots.txt / 404 / プライバシーポリシー / 画像のWebP自動変換 /
ビルド時シンタックスハイライト / ヘッダーのSNSリンク(X・YouTube・Booth・ほしい物リスト・お問い合わせ) /
トップへ戻るボタン

### 変更する

- **前後記事ナビ**: front matter の `prev_post_path`/`next_post_path` 手書き方式を廃止し、
  `series` フィールドによる自動ナビに変更(同一シリーズ内を日付順で自動接続)
- **RSS**: 生成は継続し、新たに head の `<link rel="alternate">` とフッターにリンクを設置
  (SEO効果はないが、フィードリーダー・自動投稿ボット向け)

### 新規

- **記事検索**: ビルド時にタイトル・タグ・説明文のJSONインデックスを出力、vanilla JSで絞り込み
- **目次(TOC)**: 記事ページに追従表示(現サイトにはない)
- **見出しアンカーリンク**
- **YouTube軽量埋め込み**(ファサード方式)

### 廃止

- Hugoショートコード全般(youtube / figure / x / ogp-link-card / link-card)
- クライアントサイドOGP取得JS + Cloudflare Pages Function + GAS API
- Tailwind CSS / PostCSS / npm依存一式
- Google Fonts の読み込み

### 保留(将来検討)

- OGP画像の自動生成(サムネイルのない記事用にタイトル入り画像をビルド時生成) — OGP処理の実装が固まってから検討
- 記事内全文検索(Pagefind) — 必要になったら後付け

## 8. URL構造

既存の記事URL(`/posts/<slug>/`)・タグURL(`/tags/<tag>/`)は**維持**する
(検索エンジンの評価と外部からの被リンクを保つため)。

## 9. 実装手順

1. 新テーマ(layouts / css / js)をゼロから実装
2. render hooks + URL→カード変換(段落単位判定)を実装
3. X oEmbed・OGP取得・画像キャッシュのビルド時パイプラインを実装
4. 既存記事をスクリプトで一括変換(§7)
5. 全35記事の表示を確認
6. Tailwind / npm / functions / Google Fonts を撤去
7. Cloudflare Pages のビルド設定を更新して切り替え
