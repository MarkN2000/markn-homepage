# サイトリニューアル仕様書

markn2000.com を一から作り直すための設計ドキュメント。
(ステータス: **確定版** / 2026-07 議論により決定)

## 1. リニューアルの目的

- 記事を**普通のMarkdownファイル**として書けるようにする(専用表記の全廃)
- スマホでも読みやすいレスポンシブデザイン
- 読み込み・動作の軽いサイト(クライアントJSを最小化)
- **何年も保守できる**構成(依存関係を最小化し、放置耐性を最大化)
- デザインは zenn.dev を参考にしたカード型・固定ライトテーマ

## 2. 技術選定(確定)

| 項目 | 選定 | 理由 |
|---|---|---|
| SSG | **Hugo** | 単一バイナリで npm 依存ゼロ。バージョン固定で何年後も同じビルドを再現可能。画像処理・ハイライト・リモート取得を内蔵 |
| CSS | **素のCSS** | Tailwind/PostCSS/npm を全廃。カスタムプロパティによるデザイントークン方式 |
| JS | **vanilla JS 最小限** | 検索絞り込み・リンクコピー・YouTube遅延読込・トップへ戻る程度。ライブラリなし |
| フォント | **システムフォント** | Google Fonts(Noto Sans JP)の読み込みを廃止。ヒラギノ / Noto Sans JP / 游ゴシックのOSスタック |
| ホスティング | Cloudflare Pages(継続) | Pages Functions(`functions/`)は廃止 |

### 保守方針

- Hugo のバージョンをリポジトリに明記して固定(CI でも同バージョンを指定)
- `package.json` / `node_modules` / `postcss.config.js` を撤廃。ビルドは Hugo バイナリ1個で完結
- README にビルド手順を1コマンドで記載

## 3. 記事フォーマット(純粋Markdown)

```yaml
---
title: "記事タイトル"
date: 2026-01-01T00:00:00+09:00
tags: ["Resonite", "作ったもの"]
description: "OGP・一覧に使う説明文"
thumbnail: "thumbnail.webp"   # 任意
emoji: "⚡"                    # 任意。thumbnail がない場合のカバー絵文字
series: "アバター解説"          # 任意。シリーズ記事の前後ナビを自動生成(§6)
externalUrl: "https://…"      # 任意。一覧カードのリンク先を外部サイトにする(公開ツール記事用)
pickup: true                  # 任意。トップページのピックアップ枠に掲載
draft: false
---
```

### URL自動変換の規則(このサイトの中核仕様)

**前後に空行のある、行に単独で書かれたURL**(=Markdownとして独立した段落のURL)だけをビルド時に変換する。GitHub/Zennと同じ感覚。

| URLの種類 | 変換結果 |
|---|---|
| YouTube | 軽量埋め込み(サムネ+再生ボタンのみ先行表示。クリックで本物のプレイヤーを読込) |
| X(旧Twitter) | ビルド時に oEmbed で本文取得 → 静的な引用カード。取得失敗時は「Xでポストを見る」シンプルカードにフォールバック |
| その他 | OGPリンクカード(§5) |

- 文中に書いたURL・`[テキスト](URL)` 形式のリンクは変換しない(普通のリンクのまま)
- キャプション付き画像は標準記法 `![alt](画像 "キャプション")` → `<figure>`+`<figcaption>`
- 生のHTML(`<br>` 等)はMarkdownの仕様どおりそのまま使える(乱用はしない)
- GFMアラート・読了時間表示は**採用しない**(必要になったら後付け可能)

## 4. デザイン(本章の記述が正)

デザインは 2026-07 のモックレビューを経て実装済み。**実装(`layouts/` と `assets/css/`)が正**であり、本章は設計意図の記録として残す。乖離があれば実装を優先し、必要に応じて本章を更新すること。

- **固定ライトテーマ**(ダークモードなし。ただし色はすべてカスタムプロパティで定義し将来の追加を容易に)
- 水色基調。**ページ全体の背景は現行サイトと同じタイル画像(`/images/bg.png`)をリピート**(2026-07 決定。多層CSSグラデーションでの柄描画はスクロール時の負荷が高く廃案)。柄の上に直接載るセクション見出し類は白いバーに載せて可読性を確保する
- トップ(`/`)は専用ランディングページ(2026-07 決定。構成は後述)。記事一覧は `/posts/` に移設し、検索ボックス・タグ絞り込み(件数付き)・カードグリッド(3→2→1列)・ページネーションはそちらに置く
- 記事: デスクトップは本文+追従目次の2カラム、モバイル1カラム
- カードカバー: thumbnail → emoji+グラデーション の順でフォールバック
- 見出しホバーの「#」アンカーは**設けない**(2026-07 決定。見出しIDは目次ジャンプ用に付与する)
- ヘッダーのSNSアイコンは **X・YouTube・お問い合わせの3つ**に絞る(BOOTH・ほしい物リスト含む全リンクはトップのヒーローに掲載)
- サイト名表記は**「MarkN(まーくん)」(句点なし)**に統一。ヘッダーはプロフィール画像アイコン+サイト名で、モバイルでは「MarkN」に短縮表示
- **トップページ(`/`)ではヘッダーバーを表示しない**(ヒーローがヘッダーを兼ねる。2026-07 決定)

### トップページ構成(上から順)

1. **プロフィールヒーロー**: アイコン・名乗り・一言紹介・SNSリンク。背景に現行踏襲の写真(header.webp / モバイルは header-mobile.webp)+可読性のための暗めオーバーレイ。モバイルでは縦積み・中央揃え(PCは横並び・左寄せ)
2. **ピックアップ**: frontmatter `pickup: true` の記事を新しい順に最大6件
3. **タグ棚**: 「特定タグの新着数件 + そのタグ一覧へのリンク」の棚。どのタグを何番目に出すかは hugo.toml のパラメータで設定し、テンプレート修正なしで増減・入替できる(初期は「作ったもの」を想定)
   - ピックアップ棚・タグ棚の右端に**縦型の「もっと見る」レール**(矢印・縦書きラベル・全件数)を常設(2026-07 決定)。PCではカード行の右端、モバイルでは横スワイプの末尾に同じ部品が現れ、続きがあることを示す
4. **最新記事**: 新着6件 + 記事一覧(`/posts/`)へのボタン

- お知らせ欄は設けない(2026-07 決定)
- 初期のピックアップ記事は移行時に選定する

### タイポグラフィ(可読性の設計目標)

現行サイトの実測(2026-07。本文幅816px・行間1.75・グレー文字・フォント指定が変数名ミスで不発、が判明)への対策として:

- 本文: **17px(モバイル16px) / line-height 1.9 / 文字色 --ink(ほぼ黒)**
- 本文の最大幅: **約860px**(2026-07 モック確認で700→760→860pxと段階的に拡大。記事エリア全体は目次込みで最大約1180px)
- 段落余白は上下対称に。見出し装飾はミニマルに(現行の左ボーダー+下線+疑似要素ドットのような過剰装飾をやめる)
- 記事は**タイトルカードと本文カードに分離**し、間・周囲に背景柄を見せる。モバイルでも全幅にせず左右に余白を残す(2026-07 決定)

### テーマ構造(保守性・ダークモード準備)

- CSS は tokens / base / layout / components に分割し、Hugo Pipes(`resources.Concat`)で結合・圧縮(npm 不要)
- 色・余白・文字サイズ・角丸・影はすべて意味ベースのカスタムプロパティとして tokens に集約。コンポーネントは生の色コードを持たない
- ダークモードは非対応だが、`:root[data-theme="dark"]` でトークンを再定義するだけで追加できる構造にしておく
- 全コンポーネントとトークンを一覧できる**スタイルガイドページ**を同梱する(本番ビルドには含めない)

### デザイン確認プロセス

1. 実装前に静的HTMLモック(トップ/記事/一覧)を作り、実ブラウザ(特にスマホ)で確認・承認を得てから実装する
2. 実装中はスマホ/タブレット/PC幅のスクリーンショットを節目ごとに共有する
3. スタイルガイドページで変更の影響を一覧確認する

## 5. OGPリンクカード(ビルド時取得・画像も自前配信)

- ビルド時に Hugo の `resources.GetRemote` でリンク先のOGP情報を取得し静的HTML化
- **サムネイル画像もビルド時に取得 → リサイズ → WebP化して自サイトから配信**
  (GitHub の camo やはてなブログと同方式。リンク切れに強く、読者のブラウザは外部サイトへアクセスしない)
- カードの画像表示域は**正方形固定**。ビルド時に Hugo の `Fill`(Smart クロップ)で正方形WebPを生成する(帯付き縮小ではなくクロップ。2026-07 決定)
- 取得失敗時は**成功時と同一レイアウト**のまま、タイトル部にURL・画像部にプレースホルダー・ファビコン+ドメイン表示にフォールバック。**ビルドは失敗させない**
- キャッシュは永続化せず**毎ビルド再取得**する(2026-07 決定)。ローカルビルドや CI の追加を避け、push だけで完結する運用を優先。Cloudflare Pages のビルドは push 時にしか走らないため、サイト放置中にカードが勝手に劣化することはない。カードを執筆時点で凍結したくなったら、取得結果を自動コミットする GitHub Actions を後付けする選択肢を残す
- クライアントJS・`functions/api/fetch-metadata`・GAS の `ogpApiUrl` は廃止

### 変更する

- **前後記事ナビ**: front matter の `prev_post_path`/`next_post_path` 手書き方式を廃止し、
  `series` フィールドによる自動ナビに変更(同一シリーズ内を日付順で自動接続)
- **RSS**: 生成は継続し、新たに head の `<link rel="alternate">` とフッターにリンクを設置
  (SEO効果はないが、フィードリーダー・自動投稿ボット向け)。フィードは全文でなく概要(description)のみ配信

### 新規

- **記事検索**: ビルド時にタイトル・タグ・説明文のJSONインデックスを出力、vanilla JSで絞り込み(設置は**記事一覧ページ `/posts/` のみ**。インデックスJSONもそこでのみ読み込む)
- **目次(TOC)**: 見出しから自動生成。デスクトップ(1024px以上)は本文横に追従表示、それ未満の幅では**タイトルカードと本文カードの間に折りたたみ式(`<details>`)で表示**(2026-07 決定)。見出しIDへのページ内ジャンプ
- **コードブロックのコピーボタン**(vanilla JS)
- **YouTube軽量埋め込み**(ファサード方式)

### 廃止

- Hugoショートコード全般(youtube / figure / x / ogp-link-card / link-card)
- クライアントサイドOGP取得JS + Cloudflare Pages Function + GAS API
- Tailwind CSS / PostCSS / npm依存一式
- Google Fonts の読み込み

### 保留(将来検討)

- OGP画像の自動生成(サムネイルのない記事用にタイトル入り画像をビルド時生成) — OGP処理の実装が固まってから検討
- 記事内全文検索(Pagefind) — 必要になったら後付け
- アクセス解析(Cloudflare Web Analytics などクッキーレスのもの) — スクリプト1行で後付け可能。ピックアップ記事の選定材料にもなる
